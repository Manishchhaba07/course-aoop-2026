Q1. Stack vs Queue: Why does DFS use a Stack while BFS uses a Queue? What fundamental difference in exploration strategy does this create?

DFS uses a stack because it follows a last-in, first-out rule. This means it goes deep into one path before checking others. BFS uses a queue because it follows a first-in, first-out rule, so it explores all nearby cities first before going deeper. This creates a key difference: DFS explores depth-wise (like going down one road fully), while BFS explores level-wise (checking all nearby options first). This changes how paths are discovered.

Q2. Immutability Benefits: What are three key advantages of making the City class immutable? How does this help with using City objects as Map keys? 

Making the City class immutable has three main benefits. First, it is safe, because values cannot change after creation. Second, it is thread-safe, so multiple parts of the program can use it without problems. Third, it is reliable for hashing, since fields never change. This helps when using City as a key in a Map, because the hashCode stays constant and prevents lookup errors.


Q3. Path Optimality: Does DFS guarantee finding the shortest path? If not, which algorithm would? Why? 

DFS does not guarantee the shortest path because it may go deep into a longer route before checking shorter ones. It stops when it finds any path, not the best one. BFS, on the other hand, always finds the shortest path in terms of number of steps because it explores all nearby cities first. That is why BFS is better when you want the minimum number of connections between cities.

Q5. Encapsulation: Identify and explain three examples of encapsulation in your design.

Examples of encapsulation from the design are : First, the City class keeps its data private and provides getters to access it. 
Second, the TravelGraph hides its internal adj_list and only allows changes through methods like addCity and addConnection. 
Third, input validation (like null checks) is inside methods, so users cannot break the system easily. This protects data and ensures controlled access and modification.
